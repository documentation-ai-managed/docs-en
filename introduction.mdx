## Overview

En provides a centralized space to organize, manage, and publish your project documentation. You create pages, embed code examples, and collaborate with your team seamlessly. This default documentation hub scales with your projects, offering intuitive tools for version control and search.

## Key Features

En simplifies documentation management with powerful features designed for developers and teams.

## Benefits of Using En

You gain efficiency by centralizing all project docs in one place, reducing context-switching. Teams collaborate without version conflicts, and readers access up-to-date information quickly. En's components make your docs engaging and professional, improving developer experience and onboarding speed.

## Quick Start

Get up and running with En in minutes.

<Steps>

<Step title="Create Your Space" icon="plus">

  Sign up at `https://en.example.com` and initialize your default documentation space.

</Step>

<Step title="Add Your First Page" icon="file-text">

  Click "New Page" and use the MDX editor to write content.

  <CodeGroup tabs="JavaScript,Python">

  ````javascript
  // Example API call to fetch docs
  const response = await fetch('https://api.example.com/docs', {
    headers: { Authorization: `Bearer ${YOUR_API_KEY}` }
  });
  ````

  ````python
  import requests
  response = requests.get('https://api.example.com/docs', headers={
      'Authorization': f'Bearer {YOUR_API_KEY}'
  })
  ````

  </CodeGroup>

</Step>

<Step title="Publish and Share" icon="share-2">

  Preview changes, then publish. Share the link: `https://docs.example.com/introduction`.

</Step>

</Steps>
