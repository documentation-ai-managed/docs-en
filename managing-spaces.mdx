## Overview

En allows you to create dedicated documentation spaces for each project. Each space acts as an isolated environment where you organize pages, manage permissions, and customize branding. Use spaces to separate documentation for different teams, products, or clients.

## Creating a New Space

Follow these steps to create a documentation space via the dashboard.

<Steps>
  <Step title="Navigate to Spaces" icon="layout">
    Go to your En dashboard at `https://dashboard.example.com/spaces`.
  </Step>
  <Step title="Click Create" icon="plus">
    Select the **New Space** button.
  </Step>
  <Step title="Configure Basics">
    Enter a name like `My Project Docs`, select a slug such as `my-project`, and choose your brand color `#3B82F6`.
  </Step>
  <Step title="Save and Customize">
    Click **Create Space**. Add pages and set permissions next.
  </Step>
</Steps>

<Callout kind="tip">
  Customize your space with a logo and custom domain for a branded experience.
</Callout>

## Space Configuration Options

Configure spaces using the settings panel or API. Key options include:

<ParamField path="name" param-type="string" required="true">
  Human-readable name for the space.
</ParamField>

<ParamField path="slug" param-type="string" required="true">
  URL-friendly identifier, e.g., `my-project`.
</ParamField>

<ParamField path="color" param-type="string" required="false">
  Brand color in hex format, defaults to `#3B82F6`.
</ParamField>

<ParamField body="public" param-type="boolean" required="false">
  Set to `true` for public access.
</ParamField>

## Setting Permissions and Access Levels

Control access with role-based permissions. Assign roles to users or teams.

| Role       | View Pages | Edit Pages | Manage Spaces | Invite Users |
|------------|------------|------------|---------------|--------------|
| Viewer     | ✅         | ❌         | ❌            | ❌           |
| Editor     | ✅         | ✅         | ❌            | ❌           |
| Admin      | ✅         | ✅         | ✅            | ✅           |

Use the API to manage permissions programmatically.

<CodeGroup tabs="JavaScript,cURL">
  ```javascript
  const response = await fetch('https://api.example.com/spaces/my-project/permissions', {
    method: 'POST',
    headers: { 'Authorization': 'Bearer YOUR_TOKEN' },
    body: JSON.stringify({
      userId: 'user-123',
      role: 'editor'
    })
  });
  ```
  ```bash
  curl -X POST https://api.example.com/spaces/my-project/permissions \
    -H "Authorization: Bearer YOUR_TOKEN" \
    -d '{"userId": "user-123", "role": "editor"}'
  ```
</CodeGroup>

<Callout kind="alert">
  Always use the principle of least privilege. Review permissions regularly.
</Callout>

## Archiving and Exporting Spaces

<Expandable title="Archiving a Space" default-open="true">
  Archive inactive spaces to free resources without deleting content.

  <Steps>
    <Step title="Select Space">
      Go to space settings.
    </Step>
    <Step title="Archive">
      Click **Archive Space**. Reactivate anytime.
    </Step>
  </Steps>
</Expandable>

<Expandable title="Exporting Content">
  Export your space as ZIP for backups or migration.

  ```bash
  curl -X GET https://api.example.com/spaces/my-project/export \
    -H "Authorization: Bearer YOUR_TOKEN" \
    --output my-project-docs.zip
  ```
</Expandable>

<Callout kind="info">
  Exports include all pages, assets, and settings. Use for compliance or transfers.
</Callout>
